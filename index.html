영화 포스터 퀴즈

2025.03.14 - 2025.03.19

영화 포스터를 보고 어떤 영화인지 맞추는 퀴즈 서비스입니다. 사용자는 제공된 영화 포스터를 보고 해당 영화의 제목을 맞추는 방식으로 진행됩니다. 이 프로젝트는 TMDB API를 이용하여 최신 영화 포스터를 동적으로 불러오고, 이를 퀴즈 형식으로 제공합니다.

📢 주요 기능

  • 게임 시작 옵션: 10, 20, 30문제 중 선택(기본값: 10문제), 옵션 미선택 시 시작 불가
  • TMDB API를 활용해 영화 데이터를 불러와 퀴즈 출제
  • 랜덤으로 불러온 포스터가 중복되지 않도록 처리
  • 정답 입력 시 공백이나 부제 없이도 정답 처리 되도록 처리
  • 마우스 클릭 또는 Enter 키로 정답 제출
  • 페이지 로드 시 자동으로 포커스 잡기
  • 게임 종료 후 맞춘 문제 개수 표시

🛠️ 사용 기술

  • HTML: 웹 페이지의 구조와 콘텐츠를 구성하기 위한 마크업 언어
  • CSS: 웹 페이지의 스타일을 정의하고 레이아웃을 조정하는 스타일시트 언어
  • JavaScript (JS): 사용자 인터랙션과 동적인 웹 기능을 구현하기 위한 프로그래밍 언어
  • API: TMDB API (영화 데이터 및 포스터 랜덤 추출)
  • 데이터 처리: fetch() 사용하여 API 요청 및 응답 처리

🚀 트러블 슈팅

- API 데이터 구조 문제

  • 문제점: 정답 확인을 위한 포스터와 영화 제목을 함께 제공하는 API를 찾는 데 어려움이 있었음.
  • 해결: 적절한 TMDB API를 찾아서 필요한 데이터를 불러옴.
  • 회고: API 선택 시 데이터 제공 방식이 적합한지 확인하는 것이 중요함.

- 랜덤 영화 출제 방식

  • 문제점: 여러 페이지의 데이터를 가져와서 200개 중 1개를 랜덤 선택하려 했으나 배우지않은 스킬과 구현이 복잡함.
  • 해결: 페이지를 랜덤하게 선택하고, 해당 페이지 내에서 20개 중 1개를 랜덤으로 가져오는 방식으로 변경.
  • 회고: 랜덤성을 유지하면서도 구현이 간단한 방식을 선택하는 것이 중요함.

- API 요청 방식 개선

  • 문제점: 기존 XHR 방식은 코드가 길고 유지보수가 어려움.
  • 해결: 최신기술인 fetch()를 사용하여 API 요청을 간결하게 처리.
  • 회고: 최신 기술을 적극적으로 활용하면 코드의 가독성과 유지보수성이 향상됨.

- 페이지 로드 시 포커스 기능 문제

  • 문제점: 퀴즈 진행 중 입력란 포커스가 가끔 풀리는 현상 발생.
  • 해결: 원인 파악이 어려워 해결하지 못함(될 때가 있고 안될 때가 존재).
  • 회고: 포커스 유지 방법을 추가적으로 연구할 필요가 있음.

- 페이지 로드 속도 지연

  • 문제점: 인텔리제이를 재시작하면 API 데이터 로딩이 느려지고 입력란 포커스 기능이 정상 작동하지 않음.
  • 해결: 시간이 지나면 정상 속도로 복구됨. 원인 분석이 필요하지만, 즉각적인 해결책은 찾지 못함.
  • 회고: API 요청 속도를 안정적으로 유지할 수 있는 방법을 추가적으로 고민할 필요가 있음.

학원 강의

타자(TAZA) | 택시 호출

2025.03.12 - 2025.03.14

이 프로젝트는 카카오 개발자 API를 활용하여 사용자에게 편리한 택시 호출 서비스를 제공합니다. 출발지와 도착지를 손쉽게 선택하고, 카카오 지도와 길찾기 기능을 통해 최적의 경로를 안내받으며 택시를 호출할 수 있습니다. 또한, 호출 내역을 통해 이전에 이용한 택시 호출 내역을 확인할 수 있어, 사용자는 더 나은 서비스를 경험할 수 있습니다. 이번 프로젝트를 통해 사용자 경험을 더욱 향상시키고, 다양한 API 연동을 통해 실생활에 유용한 서비스를 구현할 수 있었습니다. 이 프로젝트를 바탕으로 향후 더 많은 기능을 추가하고, 사용자 맞춤형 서비스를 제공할 수 있는 기회를 기대합니다.

📢 주요 기능

  • 카카오 지도 API를 활용하여 사용자가 지도에서 위치를 살펴볼 수 있습니다.
  • 카카오 길찾기 API를 이용해 출발지와 도착지 주소를 손쉽게 찾을 수 있습니다.
  • 출발지와 도착지를 선택하면, 최적의 길찾기 기능을 통해 경로가 선으로 표시되며, 예상 금액도 함께 안내됩니다.
  • 카카오 개발자 API를 활용하여 결제 시스템을 연동, 카카오 QR코드를 이용해 가상 결제도 지원합니다.
  • 호출 내역 기능을 통해 이전에 호출했던 택시 내역을 확인하고, 사용자가 빠르게 접근할 수 있도록 제공합니다.
  • 이 프로젝트는 카카오의 다양한 API를 통합하여 실용적이고 직관적인 택시 호출 서비스를 제공합니다.

🛠️ 사용 기술

  • HTML: 웹 페이지의 구조와 콘텐츠를 구성하기 위한 마크업 언어
  • CSS: 웹 페이지의 스타일을 정의하고 레이아웃을 조정하는 스타일시트 언어
  • JavaScript (JS): 사용자 인터랙션과 동적인 웹 기능을 구현하기 위한 프로그래밍 언어
  • API: 카카오 개발자, 카카오 지도, 카카오 길찾기, 카카오 주소 찾기
  • XMLHttpRequest (XHR): 서버와 비동기식 데이터 송수신을 통해 페이지 업데이트 및 데이터 처리

🚀 트러블 슈팅

  • 문제점: 이번 프로젝트는 강사님이 만든 템플릿을 바탕으로 진행했기 때문에, 큰 기술적 문제나 트러블슈팅 이슈는 없었습니다. 다만, 프로젝트를 진행하면서 일부 기능을 이해하고 적용하는 데 있어 시간이 필요했습니다. 문제 해결보다는 기능에 대한 이해와 공부에 중점을 두었습니다.
  • 회고: 이 프로젝트는 카카오의 다양한 API를 활용하여 택시 호출 서비스를 구현하는 과정에서 실용적인 서비스를 제공할 수 있었습니다. 사용자가 지도와 길찾기, 결제 시스템을 자연스럽게 경험할 수 있도록 UI/UX에 신경을 썼고, 기능들을 잘 통합할 수 있었습니다. 다만, 처음 접하는 카카오 개발자 API와 통합 작업이었기 때문에, 문서를 꼼꼼히 읽고 테스트하는 과정에서 많이 배울 수 있었고, 좀 더 다가갈 수 있는 시간이 되었습니다.
